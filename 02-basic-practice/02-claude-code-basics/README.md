# 02. Claude Code 기초

> 파일을 만들고, 읽고, 수정하는 기본 패턴을 익힌다

---

## Claude Code로 할 수 있는 것

| 기능 | 예시 지시 |
|------|-----------|
| 파일 만들기 | "index.html 파일을 만들어줘" |
| 파일 읽고 요약 | "report.docx를 요약해줘" |
| 여러 파일 처리 | "이 폴더의 모든 .txt 파일을 하나로 합쳐줘" |
| 파일 수정 | "이 파일에서 회사명을 전부 바꿔줘" |
| 웹에서 정보 수집 | "경쟁사 홈페이지를 방문해서 주요 서비스를 정리해줘" |
| 분석 및 보고서 | "sales.csv를 분석해서 보고서를 만들어줘" |

---

## 실습 1: 파일 만들기

```bash
mkdir practice-1
cd practice-1
claude
```

입력:
```
이번 달 업무 계획을 작성하는 마크다운 템플릿 파일을 만들어줘.
파일명: work-plan.md
포함 항목: 이번 달 목표, 주간 계획표, 완료 체크리스트
```

---

## 실습 2: 폴더 분석하기

기존 폴더를 Claude Code에서 열고:

```
이 폴더에 어떤 파일들이 있는지 확인하고
파일 목록과 각 파일의 내용을 간단히 요약해줘.
```

---

## 실습 3: 데이터 정리

CSV 파일이나 텍스트 파일이 있다면:

```
[파일명] 파일을 열어서 내용을 분석해줘.
중복된 항목이 있으면 제거하고
[기준]별로 정렬해서 새 파일로 저장해줘.
```

---

## 실습 4: 일괄 변환

폴더에 파일이 여러 개 있을 때:

```
이 폴더에 있는 모든 마크다운(.md) 파일을 읽어서
각 파일의 제목과 첫 번째 단락을 모아
"목차.md" 파일로 만들어줘.
```

---

## CLAUDE.md: Claude에게 규칙 주기

프로젝트 폴더에 `CLAUDE.md` 파일을 만들면
Claude Code가 항상 그 내용을 기억합니다.

**CLAUDE.md 예시 (마케팅 팀):**

```markdown
# 마케팅팀 AI 작업 규칙

## 브랜드 톤
- 항상 친근하고 전문적인 톤 유지
- 과도한 형식어 지양

## 파일 규칙
- 모든 파일은 한국어로 작성
- 날짜 형식: YYYY-MM-DD
- 저장 폴더: output/

## 금지 사항
- 개인정보가 포함된 파일 직접 수정 금지
- 원본 파일 삭제 금지 (항상 새 파일로 저장)
```

---

## 안전하게 사용하기

**중요 파일은 작업 전에 백업:**

```bash
# 폴더 전체 복사 (Mac/Linux)
cp -r 원본폴더 원본폴더_backup

# Windows
xcopy 원본폴더 원본폴더_backup /E /I
```

**결과는 항상 확인:**
- 숫자(금액, 날짜, 수량) 특히 주의
- 중요 문서는 사람이 최종 검토
