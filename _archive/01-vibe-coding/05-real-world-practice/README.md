# 05. 실전 연습

> 내 업무에 바이브 코딩 적용하기

---

## 직무별 활용 시나리오

직접 해보고 싶은 시나리오를 골라보세요.

---

### 시나리오 A: 팀 내부 도구 만들기 (운영/기획)

**상황**: 팀원들이 자주 묻는 질문이 반복되어 FAQ 페이지가 필요함

Claude에게:
```
우리 팀 내부용 FAQ 웹 페이지를 만들어줘.
- 검색 기능 포함
- 카테고리: 인사 규정, IT 지원, 업무 프로세스
- 질문을 클릭하면 답변이 펼쳐지는 방식
- 예시 질문 5개를 넣어줘
```

→ 결과 HTML 파일을 팀 공유 드라이브에 올리면 즉시 활용 가능

---

### 시나리오 B: 현황 대시보드 만들기 (영업/마케팅)

**상황**: 경영진 보고를 위한 KPI 현황 페이지 필요

```
월별 KPI를 보여주는 대시보드 웹 페이지를 만들어줘.
지표: 월 매출, 신규 고객 수, 고객 유지율
샘플 데이터를 사용해서 먼저 보여주고,
나중에 내가 숫자를 직접 바꿀 수 있게 해줘.
```

---

### 시나리오 C: 자동화 문서 만들기 (HR/총무)

**상황**: 매달 같은 형식의 공지문을 작성해야 함

```
매월 보내는 급여 명세서 안내 이메일 템플릿을 만들어줘.
항목:
- 이번 달 급여 지급일
- 변경사항 공지 자리
- 문의처 안내

Word 파일(.docx)로 만들어줘.
```

---

### 시나리오 D: 이벤트 소개 페이지 만들기 (마케팅)

**상황**: 다음 달 이벤트를 홍보하는 랜딩 페이지가 급하게 필요

```
우리 회사 할인 이벤트 랜딩 페이지를 만들어줘.
이벤트명: [이름]
기간: [날짜]
혜택: [내용]
색상: 브랜드 컬러 [색상] 계열로
CTA 버튼: "지금 신청하기"
```

---

### 시나리오 E: 분석 보고서 만들기 (기획/전략)

**상황**: 경쟁사 3곳을 비교 분석한 보고서가 필요

```
경쟁사 분석 보고서 초안을 작성해줘.
비교 대상: A사, B사, C사
비교 항목: 주요 서비스, 가격대, 강점/약점, 타깃 고객
표 형태로 먼저 정리하고, 아래에 분석 코멘트를 추가해줘.
```

---

## 결과물을 실제로 활용하기

바이브 코딩으로 만든 것을 그냥 두지 마세요.

```
1단계: 만들기 (바이브 코딩으로 빠르게)
2단계: 써보기 (실제 업무에 사용)
3단계: 피드백 (부족한 점 파악)
4단계: 개선하기 ("이 부분을 이렇게 바꿔줘")
```

처음부터 완벽할 필요 없습니다.
쓰면서 고치는 것이 바이브 코딩의 핵심입니다.

---

## 막힐 때 쓰는 프롬프트

### 결과가 마음에 안 들 때
```
결과가 내가 원하는 것과 달라.
내가 원하는 건 [구체적 설명]이야.
다시 만들어줘.
```

### 어떻게 시작해야 할지 모를 때
```
나는 [직무]에서 일해.
[업무 설명]을 도와주는 간단한 도구를 만들고 싶은데
어떻게 시작하면 좋을지 제안해줘.
```

### 결과를 더 다듬고 싶을 때
```
만든 것의 디자인을 더 전문적으로 바꿔줘.
색상과 글꼴을 세련되게 수정해줘.
```
