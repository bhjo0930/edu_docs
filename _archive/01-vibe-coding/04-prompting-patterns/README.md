# 04. 프롬프트 패턴

효과적으로 AI에게 지시하는 방법들을 배웁니다.

---

## 핵심 원칙

> **"AI는 당신의 의도를 추측한다. 명확하게 말할수록 정확해진다."**

---

## 패턴 1: 컨텍스트 + 요청

**나쁜 예:**
```
버튼 만들어줘
```

**좋은 예:**
```
[컨텍스트] 쇼핑몰 랜딩 페이지를 만들고 있어.
[요청] 구매하기 버튼을 만들어줘.
[조건] 빨간색, 크기는 크게, 클릭하면 /cart 페이지로 이동.
```

---

## 패턴 2: 예시 제공 (Few-shot)

```
아래 형식으로 카드 3개를 만들어줘:

예시 카드:
- 제목: "빠른 배송"
- 아이콘: 🚀
- 설명: "주문 후 24시간 이내 배송"

만들 카드:
1. 안전한 결제 (아이콘: 🔒)
2. 쉬운 반품 (아이콘: ↩️)
3. 고객 지원 (아이콘: 💬)
```

---

## 패턴 3: 단계적 구체화 (Iterative)

```
1단계: "간단한 포트폴리오 페이지 만들어줘"
   ↓ (결과 확인)
2단계: "헤더 섹션에 내 이름과 직함을 크게 표시해줘"
   ↓ (결과 확인)
3단계: "프로젝트 섹션을 카드 그리드 레이아웃으로 바꿔줘"
   ↓ (결과 확인)
4단계: "다크 모드 토글 버튼 추가해줘"
```

한 번에 모든 걸 요청하지 말고, **작게 나눠서 확인하며 진행**.

---

## 패턴 4: 역할 부여 (Role)

```
너는 UX 디자이너야.
현재 내 랜딩 페이지를 보고 개선점 3가지를 제안해줘.
그리고 그 중 하나를 직접 구현해줘.
```

```
시니어 백엔드 개발자 입장에서
이 코드의 보안 취약점을 찾아줘.
```

---

## 패턴 5: 제약 조건 명시

```
다음 조건으로 네비게이션 메뉴를 만들어줘:
- 외부 라이브러리 사용 금지 (순수 HTML/CSS/JS만)
- 모바일 반응형 필수
- 애니메이션 없음
- 메뉴 항목: 홈, 소개, 포트폴리오, 연락처
```

---

## 패턴 6: 에러 해결

```
아래 에러가 발생했어. 원인과 해결 방법을 알려줘.

[에러 메시지]
TypeError: Cannot read property 'map' of undefined
  at App.js:15

[관련 코드]
const items = data.items;
items.map(item => ...)

[상황]
API에서 데이터를 가져오는 중에 발생함
```

---

## 패턴 7: 비교 요청

```
같은 기능을 하는 코드를 2가지 방법으로 만들어줘:
1. CSS Grid 사용
2. CSS Flexbox 사용

그리고 각각 언제 쓰는 게 좋은지 설명해줘.
```

---

## 안티패턴 (하지 말 것)

| 안티패턴 | 이유 | 대안 |
|----------|------|------|
| 한 번에 모든 기능 요청 | AI가 방향을 잃음 | 기능을 나눠서 단계적으로 |
| "더 좋게 만들어줘" | 기준이 없음 | "로딩 속도를 개선해줘" |
| 에러만 붙여넣기 | 컨텍스트 부족 | 에러 + 코드 + 상황 설명 |
| 결과 확인 없이 계속 요청 | 방향이 틀릴 수 있음 | 각 단계 결과 반드시 확인 |
