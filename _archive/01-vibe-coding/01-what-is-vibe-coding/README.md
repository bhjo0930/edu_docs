# 01. 바이브 코딩이란?

---

## 1. 등장 배경

2025년 초, Andrej Karpathy(전 OpenAI, Tesla AI 디렉터)가 트위터에 올린 한 마디:

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."

번역하면: *"코드가 존재한다는 사실조차 잊어버리고, 그냥 느낌(vibe)에 따라 AI에게 맡기는 새로운 코딩 방식"*

이 개념이 폭발적으로 퍼진 이유: **실제로 동작하기 때문.**

---

## 2. 전통적 코딩 vs 바이브 코딩

| 구분 | 전통적 코딩 | 바이브 코딩 |
|------|------------|------------|
| 주체 | 개발자가 직접 코드 작성 | AI가 코드 생성, 사람은 방향 제시 |
| 필요 지식 | 언어 문법, 알고리즘, 프레임워크 | 무엇을 만들지 명확한 표현 능력 |
| 속도 | 아이디어 → 구현 (일/주 단위) | 아이디어 → 구현 (시간 단위) |
| 병목 | 구현 능력 | 아이디어와 검토 능력 |
| 결과물 검증 | 코드 리뷰 | 동작 확인 + 가이드 리뷰 |

---

## 3. 바이브 코딩의 핵심 루프

```
아이디어 떠오름
      ↓
자연어로 AI에게 설명
      ↓
AI가 코드/결과물 생성
      ↓
결과 확인 (동작하나?)
      ↓
[안 됨] → 문제를 AI에게 설명 → 다시 생성
[됨]    → 다음 기능으로 이동
```

**핵심: 코드를 읽으려 하지 말고, 결과를 보라.**

---

## 4. 무엇을 만들 수 있나?

### 빠르게 만들 수 있는 것들

- **정적 웹사이트**: 포트폴리오, 랜딩 페이지, 소개 페이지
- **간단한 웹앱**: To-do 앱, 계산기, 폼 처리
- **데이터 시각화**: 차트, 대시보드 프로토타입
- **자동화 스크립트**: 파일 처리, 데이터 변환, 반복 작업
- **API 프로토타입**: 기본적인 CRUD 서버

### 바이브 코딩의 한계

- 복잡한 비즈니스 로직 (검토 능력 필요)
- 보안이 중요한 시스템 (전문가 검토 필요)
- 대규모 팀 협업 코드베이스 (구조 설계 필요)

---

## 5. 왜 지금 배워야 하나?

```
2023: "AI가 보조 도구"
2024: "AI가 주요 코딩 파트너"
2025: "AI 없이 코딩하는 게 비효율"
2026: "AI와 협업 못하면 생산성 격차 10x"
```

바이브 코딩은 기술이 아니라 **마인드셋**의 변화입니다.
코드를 모르는 사람도 AI를 통해 소프트웨어를 만들 수 있는 시대가 이미 왔습니다.
