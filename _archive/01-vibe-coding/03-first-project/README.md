# 03. 첫 번째 프로젝트: 랜딩 페이지 만들기

> 바이브 코딩으로 실제 결과물을 만드는 첫 경험

---

## 프로젝트 목표

Claude Code를 사용해서 **개인 소개 랜딩 페이지**를 만듭니다.
코드를 직접 작성하지 않고, AI와 대화만으로 완성합니다.

---

## 완성 목표물

```
┌─────────────────────────────────┐
│  [네비게이션: 홈 | 소개 | 연락]   │
├─────────────────────────────────┤
│                                 │
│     안녕하세요, 저는 [이름]입니다  │
│     [직함/역할]                  │
│     [CTA 버튼: 더 알아보기]       │
│                                 │
├─────────────────────────────────┤
│  [소개 섹션]                     │
├─────────────────────────────────┤
│  [연락처 섹션]                   │
└─────────────────────────────────┘
```

---

## 단계별 실습

### Step 1: 프로젝트 시작

```bash
mkdir landing-page
cd landing-page
claude
```

### Step 2: 기본 구조 생성

```
간단한 개인 소개 랜딩 페이지를 만들어줘.
- HTML, CSS, JavaScript 사용
- 네비게이션, 히어로 섹션, 소개, 연락처 포함
- 반응형 디자인
```

→ `index.html` 생성 확인 후 브라우저에서 열어보기

### Step 3: 내용 수정

```
히어로 섹션의 이름을 "[본인 이름]"으로,
직함을 "[본인 직함]"으로 바꿔줘.
```

### Step 4: 디자인 개선

```
색상 테마를 [파란색/보라색/초록색] 계열로 바꿔줘.
폰트도 더 현대적으로 변경해줘.
```

### Step 5: 인터랙션 추가

```
스크롤하면 네비게이션이 고정되도록 만들어줘.
그리고 각 섹션이 스크롤로 등장할 때
페이드인 애니메이션을 추가해줘.
```

### Step 6: 최종 점검

```
전체 페이지를 검토하고:
1. 모바일에서 잘 보이는지 확인해줘
2. 색상 대비가 접근성 기준을 충족하는지 확인해줘
3. 개선이 필요한 부분이 있으면 수정해줘
```

---

## 흔한 문제와 해결

### "원하는 대로 안 됐을 때"

```
방금 만든 [섹션명]이 내가 원하는 것과 달라.
내가 원하는 건: [구체적 설명]
지금 있는 건: [현재 상태 설명]
수정해줘.
```

### "에러가 났을 때"

```
브라우저 콘솔에서 이 에러가 나와:
[에러 메시지 붙여넣기]
고쳐줘.
```

- [ ] 네비게이션 작동
- [ ] 모바일에서 보기 좋음
- [ ] 내 이름/정보가 들어있음
- [ ] 색상/디자인이 마음에 듦
