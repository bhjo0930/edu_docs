# 02. 바이브 코딩

> "코딩을 몰라도 된다. AI에게 원하는 것을 말하면 만들어준다."

---

## 바이브 코딩이란?

**바이브 코딩(Vibe Coding)** = AI와 자연어 대화만으로 소프트웨어를 만드는 방식

2025년 Andrej Karpathy(전 OpenAI 연구 디렉터)가 명명.

핵심: **"어떻게 만드는지(How)"** 는 AI가 처리하고, **"무엇을 만들지(What)"** 는 사람이 결정한다.

---

## 기존 방식과 비교

| | 전통적 개발 | 바이브 코딩 |
|--|------------|------------|
| 필요 역량 | 프로그래밍 언어, 알고리즘 | 원하는 것을 명확히 설명하는 능력 |
| 개발 속도 | 아이디어 → 결과물: 수일~수주 | 아이디어 → 결과물: 수분~수시간 |
| 검토 방식 | 코드 리뷰 | 결과물이 제대로 동작하는지 확인 |
| 주체 | 개발자가 직접 코드 작성 | AI가 코드 생성, 사람은 방향 제시 |

---

## 바이브 코딩의 핵심 루프

```
1. 아이디어 떠오름
         ↓
2. AI에게 자연어로 설명
   ("이런 걸 만들어줘")
         ↓
3. AI가 결과물 생성
         ↓
4. 결과 확인
   → 됐다: 다음 기능으로
   → 안 됐다: 문제를 AI에게 설명하고 수정 요청
         ↓
5. 반복
```

**핵심**: 코드를 이해하려 하지 말고, **결과물이 원하는 대로 작동하는지**만 확인한다.

---

## 무엇을 만들 수 있나?

### 비개발자도 바로 만들 수 있는 것

- **웹 페이지**: 개인 소개, 회사 소개, 이벤트 랜딩 페이지
- **내부 도구**: FAQ 페이지, 신청 폼, 현황 대시보드
- **자동화**: 반복적인 파일 처리, 데이터 정리 스크립트
- **프로토타입**: 앱 화면 시안, 인터랙티브 데모

### 실제 사례

**기획자의 바이브 코딩:**
```
"경쟁사 분석 표가 있는 웹 페이지 만들어줘.
 회사별로 탭으로 나눠서 볼 수 있게."
→ 30분 안에 완성
```

**HR 담당자의 바이브 코딩:**
```
"신입사원 온보딩 체크리스트 웹 페이지 만들어줘.
 완료 체크하면 저장되고, 진행률이 표시되게."
→ 1시간 안에 완성
```

**마케터의 바이브 코딩:**
```
"이번 시즌 프로모션 랜딩 페이지 만들어줘.
 카운트다운 타이머랑 신청 폼 포함해서."
→ 45분 안에 완성
```

## 좋은 지시 방법

### 원칙: 구체적일수록 좋다

```
모호한 지시:  "보기 좋게 만들어줘"

구체적 지시:  "배경은 흰색, 텍스트는 짙은 회색,
               버튼은 파란색으로 만들어줘.
               폰트는 깔끔한 산세리프 스타일로."
```

### 단계별로 요청하기

한 번에 모든 것을 요청하면 방향이 흐트러집니다.

```
1단계: "기본 구조 먼저 만들어줘"
       → 확인 후
2단계: "헤더 디자인 수정해줘"
       → 확인 후
3단계: "모바일에서도 잘 보이게 수정해줘"
```
