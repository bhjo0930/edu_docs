# 03. AI 비서(Agent)

> "목표를 주면 스스로 처리하는 AI"

---

## 바이브 코딩 vs AI 비서

바이브 코딩과 AI 비서는 다릅니다.

| | 바이브 코딩 | AI 비서(Agent) |
|--|------------|----------------|
| 방식 | 요청 → 결과 (1회) | 목표 → 계획 → 실행 → 결과 |
| 파일 접근 | 결과를 복사해서 쓰는 방식 | 내 컴퓨터 파일에 직접 접근 |
| 작업 규모 | 단일 결과물 생성 | 여러 단계로 이어지는 복합 작업 |
| 사람 역할 | 매 단계 지시 | 목표만 주면 됨 |

---

## AI 비서가 일하는 방식

```
[사람] "이번 달 팀 성과를 정리한 보고서 만들어줘"
          ↓
[AI 비서] "알겠습니다. 이렇게 진행할게요:"

  1단계: 공유 폴더에서 이번 달 업무 보고서 파일 찾기
  2단계: 각 파일을 읽고 팀원별 성과 추출
  3단계: 통합 보고서 형식으로 정리
  4단계: "성과보고서_2026년3월.docx" 파일로 저장

  [자동으로 실행 중...]

  완료: 보고서를 저장했습니다. 검토해주세요.
```

## 일상 업무에서의 활용

### 시나리오 1: 회의록 일괄 처리

```
지시: "올해 모든 회의록에서
       결정 사항과 담당자를 추출해서
       하나의 표로 정리해줘"

AI 비서:
  - 회의록 폴더의 파일 30개 자동 처리
  - 각 파일에서 결정 사항 추출
  - 담당자별로 정리
  - 결과 파일 저장
```

### 시나리오 2: 주간 경쟁사 모니터링

```
지시: "경쟁사 3곳의 웹사이트를 방문해서
       새로운 서비스나 가격 변경이 있는지 확인하고
       변경사항을 정리해줘"

AI 비서:
  - 3개 웹사이트 자동 방문
  - 지난주와 비교
  - 변경사항 정리 보고서 생성
```

### 시나리오 3: 이메일 초안 대량 작성

```
지시: "고객 목록 파일을 보고
       각 고객의 업종에 맞는
       맞춤형 제안 이메일을 만들어줘"

AI 비서:
  - 고객 목록 100건 읽기
  - 업종별 맞춤 이메일 생성
  - 파일로 저장
```

## AI 비서의 한계

솔직하게 알아야 할 것들:

| 한계 | 내용 |
|------|------|
| 실수 가능 | 항상 결과를 검토해야 함 |
| 할루시네이션 | 없는 내용을 만들어낼 수 있음 |
| 비용 | 많은 작업 = 더 많은 비용 |
| 맥락 한계 | 매우 큰 데이터는 처리 어려움 |

**핵심 규칙**: 중요한 결과물은 반드시 사람이 검토

## AI 비서를 쓰려면?

현재 주요 AI 비서 도구:

| 도구 | 특징 | 접근 방법 |
|------|------|-----------|
| **Claude Code** | 파일 처리, 분석에 강함 | 설치 필요 (기초 실습) |
| **ChatGPT (Operator)** | 웹 브라우징, 범용 | 유료 플랜 |
| **Perplexity** | 실시간 웹 검색 특화 | 무료/유료 |

기초 실습에서 Claude Code를 직접 설치하고 써봅니다.
