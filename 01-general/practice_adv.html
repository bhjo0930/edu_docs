컨텍스트 관리 전략

컨텍스트란?

AI가 한 세션에서 기억할 수 있는 전체 대화·코드·문서의 양

Claude: ~200K 토큰 ≈ 약 150,000단어
→ 방대하지만 무한하지 않다

핵심 관리 원칙

CLAUDE.md에 프로젝트 규칙·배경 상시 기술
/clear로 불필요한 컨텍스트 주기적 정리
compact summary로 핵심만 압축 요약

Best
CLAUDE.md에 프로젝트 목적·기술스택·금지사항 명시 → 매 세션 자동 로드
Tip
긴 작업 전 /clear 실행 → 이전 실수·노이즈 제거 후 새 시작
핵심
컨텍스트 품질이 결과 품질을 결정 — 많이 넣는 것보다 정확하게 넣는 것이 중요

Context Rot — 컨텍스트 부패

대화가 길어질수록 AI의 추론 품질이 점진적으로 저하되는 현상
— 초반의 날카로운 답변이 후반에 무뎌지고 반복·오류가 증가

증상
Context Rot 신호
• 앞서 정한 규칙을 갑자기 어김
• 같은 실수가 반복됨
• 답변이 점점 일반적으로 변함
• 이전 파일과 일관성 깨짐
원인
왜 발생하는가?
토큰 한계 접근 시 초기 지시 희석
오류 메시지·반복 수정으로 노이즈 증가
모델이 최근 대화에 더 가중치를 둠
대응 1
세션 리셋 & 요약
/clear 후 핵심 진행 상황만 붙여넣기
"지금까지 완성된 것: ..., 다음 할 것: ..."
대응 2
작업 단위 분리
큰 작업을 독립적 서브태스크로 분리
각 태스크를 새 세션에서 처리
→ 각 세션의 컨텍스트를 항상 신선하게 유지

OpenClaw — 개인용 AI 에이전트

핵심 특징

• 내 컴퓨터에서 직접 실행 — 데이터 외부 유출 없음
• WhatsApp·Telegram·Slack·Discord 등 채팅앱에서 조작
• macOS·Windows·Linux 모두 지원
• 오픈소스 (MIT 라이선스) — 직접 수정 가능

자동화 가능한 업무

• 이메일 발송·정리
• 캘린더 일정 관리
• 파일 읽기·쓰기·검색
• 셸 명령 실행
• 항공편 체크인·예약 등 웹 자동화

지속 학습
영구 메모리로 사용자 선호도·패턴을 학습 → 쓸수록 더 똑똑해짐
활용 팁
Slack에서 "오늘 받은 메일 요약해줘" → 백그라운드에서 처리 후 결과 전송
openclaw.ai · 커뮤니티 스킬·플러그인으로 기능 확장 가능

harness.io · VMware·Citi·United Airlines 등 사용 중
CI/CD 자동화
Continuous Delivery & GitOps
AI가 파이프라인을 분석해 병목 제거·배포 최적화. 실패 시 자동 롤백.
AI 테스트
자가 치유 테스트
UI 변경 시 테스트 코드를 AI가 자동 수정. 테스트 유지 비용 대폭 감소.
보안
AI 위협 탐지

Paperclip — AI 에이전트 회사

핵심 비유

OpenClaw이 직원이라면,
Paperclip은 회사다

여러 AI 에이전트(OpenClaw·Claude Code·Codex 등)를 조직도·목표·예산·역할로 관리하는 오케스트레이션 플랫폼

오픈소스 · paperclip.inc · GitHub 14.2K stars

주요 기능

• 조직도·보고 라인 설정 — 에이전트가 "무엇을, 왜" 알고 실행
• 에이전트별 월간 예산 — 한도 도달 시 자동 중단
• Heartbeat 스케줄 — 정기적 자동 작업 실행
• 모든 작업 감사 로그 · 모바일 관리

시작
npx paperclipai onboard --yes → 에이전트 팀 구성 시작

MCP 사용 시

AI가 실시간으로 DB 조회·수정, 파일 읽기, 슬랙 메시지 전송 등을 자율적으로 처리

Stitch — MCP 연결 허브

여러 MCP 서버를 하나의 인터페이스로 통합 관리하는 플랫폼
— 개별 MCP 서버를 일일이 설정하지 않고 Stitch 하나로 연결

주요 기능
MCP 서버 통합 관리
GitHub, Notion, Slack, DB 등 다양한 MCP 서버를 한 곳에서 연결·관리
장점
설정 단순화
각 도구별 MCP 설정 없이 Stitch 계정 연결만으로 수십 개의 통합 사용 가능
실습 시나리오
GitHub + Notion 연동
"오늘 머지된 PR을 정리해서 Notion 업무 일지에 자동 기록해줘"
설정 위치
.mcp.json 파일
프로젝트 루트의 .mcp.json에 Stitch 서버 URL 등록 → Claude Code가 자동 인식
2부

Supabase MCP — AI가 DB를 직접 다룬다

설정
.mcp.json에 Supabase MCP 서버 등록 → Project URL + API 키 입력
사용
Claude Code에게 자연어로 DB 작업 지시 — 테이블 조회, 데이터 삽입, 스키마 수정
예시
"users 테이블에서 이번 달 신규 가입자 수를 지역별로 집계해서 보여줘"
할 수 있는 것

• 테이블 생성·수정·삭제
• SQL 쿼리 자동 생성·실행
• 데이터 분석 및 시각화 코드 생성
• Row Level Security 정책 설정

주의 사항

• 프로덕션 DB에 직접 연결 주의
• 삭제·수정 전 반드시 확인 요청
• 개발/스테이징 환경 우선 사용
• API 키 .gitignore 처리 필수

Skills란? 구조와 작동 방식

Skills 없이

매번 같은 맥락·지시를 처음부터 입력
"코드 리뷰할 때마다 동일한 기준 설명"
→ 시간 낭비, 일관성 부족

Skills 사용 시

/code-review 입력 한 번
→ 사전 정의된 기준으로 즉시 리뷰 시작
→ 팀 전체가 동일한 기준 공유

Skills 파일 구조 — .claude/commands/code-review.md
--- name: code-review description: 팀 코딩 컨벤션 기준으로 코드 리뷰 --- # 리뷰 기준 1. 변수명이 의미를 명확히 전달하는가? 2. 에러 처리가 적절한가? 3. 불필요한 중복 코드는 없는가?

Agent Team이란? 왜 팀인가?

단일 에이전트 한계

복잡한 작업을 혼자 처리 → 컨텍스트 과부하
모든 도메인을 하나의 AI가 담당 → 전문성 희석
병렬 처리 불가 → 순차 실행으로 느림

Agent Team 장점

전문화: 각자의 역할·도구에 집중
병렬: 독립 작업 동시 처리로 속도 향상
검증: 에이전트 간 상호 검토로 품질 향상

오케스트레이션 3가지 패턴

패턴 01 — Sequential
순차 파이프라인
A → B → C 순서로 결과를 넘기며 처리
각 단계가 이전 단계 결과에 의존할 때
예: 수집 → 분석 → 보고서
패턴 02 — Parallel
병렬 처리
독립적인 작업을 동시에 여러 에이전트가 처리
속도 최대화, 단 결과 통합 로직 필요
예: 다국어 번역 동시 처리
패턴 03 — Evaluator
생성-평가 루프
Generator가 결과 생성 → Evaluator가 품질 검증 → 기준 미달 시 재생성
PDCA iterate 패턴의 핵심
Claude Code 적용
Subagent 활용
Agent tool로 서브에이전트 실행
각 서브에이전트에 특화된 CLAUDE.md 적용
메인 에이전트가 결과 취합·조율
