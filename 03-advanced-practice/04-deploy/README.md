## 배포 방법 선택

| 상황 | 방법 |
|------|------|
| 팀 내부에서만 쓸 때 | 로컬 스크립트 + 스케줄러 |
| 웹으로 제공할 때 | Vercel / Render 무료 배포 |
| 기업 환경 | 내부 서버 또는 클라우드 |

---

## 방법 1: 스케줄러로 자동 실행 (권장)

매일 정해진 시간에 자동 실행합니다.

### Mac (cron)

```bash
# crontab 편집
crontab -e

# 매일 오전 9시에 실행
0 9 * * * cd /path/to/daily-briefing && claude --no-interactive "/briefing" >> logs/briefing.log 2>&1
```

### Windows (작업 스케줄러)

1. 시작 → "작업 스케줄러" 검색
2. "작업 만들기" 클릭
3. 트리거: 매일 오전 9시
4. 동작: `claude --no-interactive "/briefing"`

## 비용 관리

### Claude API 비용 계산

Claude 비용은 처리하는 텍스트 양에 따라 결정됩니다.

**일일 브리핑 예시:**
- 입력: Slack 메시지 ~5,000 토큰 + 지시문 ~500 토큰
- 출력: 브리핑 ~1,000 토큰
- 예상 비용: **일 $0.02~0.10** (모델에 따라 다름)

**월간 비용 예시:**
- 일일 브리핑 30일: $0.6~3
- 고객 문의 100건/일: $1~5/일

### 비용 최적화

```
긴 채널 히스토리 전체를 읽지 말고
최근 24시간 메시지만 읽도록 지시:

"Slack에서 최근 24시간 이내 메시지만 수집해줘."
```
